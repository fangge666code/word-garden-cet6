# 手机端当前版本排版设计

## 目标

让手机端在现有账号与安装区域中稳定显示当前版本号，同时保持桌面端布局和更新流程不变。

## 范围

- 保留现有 `CURRENT_VERSION_NAME` 数据来源和检查更新逻辑。
- 仅调整窄屏样式，不新增独立卡片，也不改动账号、同步或学习记录。
- 登录前首页、登录后首页与设置页继续复用现有版本徽标和检查更新按钮。

## 界面行为

- 在宽度不超过 520px 的设备上，版本徽标独占一整行并居中显示。
- “检查更新”按钮排列在版本徽标下方，并保持足够的可点击宽度。
- 其他账号操作按钮可自动换行，不得覆盖、裁切或隐藏版本徽标。
- 桌面端保持现有横向布局。

## 错误与兼容性

- 如果更新清单暂时无法访问，继续使用现有提示，不影响当前版本号展示。
- 支持 Android 原生壳、手机浏览器和已安装 PWA。
- 不清除本地缓存、账号会话或学习数据。

## 验证

- 在 360px、390px 和 520px 视口验证版本徽标与检查更新按钮可见。
- 验证登录前首页、登录后首页和设置页。
- 运行现有自动化测试，确认更新检查和其他功能无回归。
