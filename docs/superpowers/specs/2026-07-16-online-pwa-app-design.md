# “词间”联网型 PWA App 设计

## 目标

把现有“词间 · CET-6 学习”网站改造成可安装到 Android 和 iPhone 主屏幕的联网型 PWA。安装后使用独立图标和接近原生 App 的全屏窗口运行，同时继续使用现有 GitHub Pages 公网地址和本地学习记录。

本次不制作应用商店安装包，也不提供完整离线词库。用户需要连接网络才能打开和学习。

## 安装形态

- App 名称使用“词间”，英文短名使用“词间”。
- 启动地址保持为 GitHub Pages 网站首页。
- 显示模式使用 `standalone`，安装后隐藏普通浏览器地址栏。
- 主题色和背景色沿用网站现有绿色、米白色视觉体系。
- 屏幕方向不强制锁定，手机竖屏和横屏均可使用。
- 应用范围限制在当前网站目录，外部链接仍交给浏览器打开。

## PWA 资源

新增 Web App Manifest，包含名称、启动地址、作用域、显示模式、颜色和图标信息。提供至少 192×192 和 512×512 的 PNG 图标，并补充适合 Android 遮罩图标的 `maskable` 声明及 iPhone 使用的 Apple Touch Icon。

图标延续现有品牌：绿色圆角底色、白色字母 W。图形保持简洁，在手机桌面小尺寸下仍清晰可辨。

页面补充 Manifest、主题色、Apple Web App 和图标相关元数据。

## 联网与 Service Worker

注册一个轻量 Service Worker，用于建立标准 PWA 安装身份和后续更新控制，但不缓存完整网页、词库或学习页面。

- 正常联网时，所有页面和资源直接从网络读取。
- 断网时不尝试展示过期学习内容，页面显示明确的“请连接网络后继续学习”提示。
- 网络恢复后提供“重新加载”按钮，返回正常网站。
- 新版本 Service Worker 安装后尽快接管页面，避免用户长期停留在旧版本。
- Service Worker 注册失败不阻止普通网页使用，只记录非阻断错误。

## 安装入口

设置页新增“安装词间 App”区域：

- 支持浏览器原生安装提示时显示安装按钮，由用户主动点击后调用浏览器安装流程。
- 已安装或浏览器不提供原生安装提示时，不显示无效按钮。
- iPhone/iPad 显示文字说明：“使用 Safari 打开，点击分享，再选择添加到主屏幕”。
- 其他不支持安装的浏览器显示简短说明，不影响网站功能。
- 不自动弹出安装提示，避免打断学习。

## 学习数据

- 继续使用当前浏览器本地存储保存个人乱序种子、学习进度和设置。
- 安装后的 PWA 使用该系统分配给网站的本地存储；不同浏览器或设备之间不自动同步。
- 用户可通过现有“导出完整备份”和“导入记录”迁移学习数据。
- CSV 学习状态导出保持不变。
- 本次不新增账号、云同步或服务器数据库。

## 更新与兼容

- 现有公网网址和 GitHub Pages 部署方式保持不变。
- Manifest 和 Service Worker 使用相对当前仓库目录的路径，兼容 `/word-garden-cet6/` 子路径部署。
- 原有桌面端和移动端网页功能必须保持可用。
- 不支持 PWA 的浏览器继续把网站当作普通网页使用。

## 验证

- Manifest 可访问且字段、路径、图标尺寸有效。
- 192×192、512×512、maskable 和 Apple Touch Icon 均能正确加载。
- 支持安装提示的浏览器能够触发安装流程。
- iPhone 安装说明只在适合的设备环境出现。
- 安装后的窗口使用独立显示模式并从正确首页启动。
- 联网时首页、学习、词库、设置、JSON 和 CSV 功能正常。
- 断网时显示联网提示，不展示可能过期的完整词库。
- Service Worker 注册失败时普通网页仍可使用。
- 自动测试、构建检查、手机尺寸页面验收和公网资源检查全部通过。
