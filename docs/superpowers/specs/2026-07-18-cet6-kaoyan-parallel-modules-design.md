# 六级与考研英语并列模块设计

## 目标

在现有六级学习产品中加入独立的“考研英语核心词汇”模块。两个模块拥有相同的学习体验、英式女声、账号同步、导入导出和自主升级能力，但词条、学习进度、每日记录、设置和当前学习位置不得互相混合。现有六级用户升级后不丢失任何记录。

## 词库

- 六级模块继续使用现有 3000 个词条和稳定 ID `cet6-xxxx`。
- 考研模块包含 3500 个核心高频词，稳定 ID 为 `ky-xxxx`。
- 考研候选词来自 MIT 授权 ECDICT 的考研考试标签，并结合 BNC、当代语料频次和词条完整度筛选；优先保留常见原形，排除专名、缩写、纯词形变化和缺少有效释义的条目。
- 每个考研词条包含英文、音标、词性、中文释义、英文例句和中文例句。
- 例句必须针对具体词义，不使用大批重复的通用占位句。

## 模块结构与界面

应用建立统一的词库注册表，每个模块提供 `id`、名称、说明、词条、发音索引和音频目录。学习引擎、词库搜索、统计、导出和发音播放器只接收当前模块，不复制两套业务代码。

登录后的首页显示“大学英语六级”和“考研英语核心词汇”两个并列卡片，包含总词数、当前进度和继续学习按钮。用户选择模块后，今日进度、学习、词库和设置页面只展示当前模块的数据。页头提供明确的模块切换入口，切换前保存当前状态，返回后继续原来的会话位置。

## 本地数据隔离与兼容

每个用户按模块使用独立存储键：六级和考研分别保存设置、乱序种子、进度、每日记录与会话。当前模块 ID 单独保存，仅用于界面选择。

升级迁移规则：旧版用户数据自动作为六级模块数据读取并复制到新的六级存储键；迁移成功后保留旧键作为安全回退，不重置数据。考研模块首次打开时创建全新的默认记录。

导入导出文件写入模块标识。导入时只允许覆盖匹配模块，避免把六级备份导入考研模块。CSV 文件名和表头明确标出模块名称。

## 云端隔离

保留原有 `user_profiles`、`word_progress`、`daily_records` 作为六级数据表，不修改旧记录。新增结构相同的 `ky_user_profiles`、`ky_word_progress`、`ky_daily_records`，并配置相同的用户外键、唯一约束、行级安全策略和授权。

客户端根据当前模块选择对应的云端类名。两个模块分别同步，登录会话共享；切换模块不会上传或合并另一模块的数据。发布时提供可重复执行的 Supabase SQL 迁移脚本。

## 发音

考研 3500 个单词使用 Kokoro-82M 的 `bf_emma` 英式女声，在构建阶段生成 16000 Hz、16 位、单声道 PCM。每 100 词一个 WAV 分块，共 35 个分块；索引和文件目录与六级完全分开。

网页版和 Android 都只按需下载当前模块当前分块并持久缓存。Android APK 只包含小型程序壳、词条和音频索引，不再内置大型音频。每个资源分块记录独立 SHA-256；未来更新只下载新增或哈希变化的文件，未变化缓存跨版本保留，废弃哈希安全清理。受 Android 安装机制限制，程序代码变化仍需安装新的轻量 APK。女声音频失败时继续回退到浏览器语音和 Android 原生 TTS。

## 验证

- 校验六级 3000 与考研 3500 个稳定 ID、词条完整度和模块内唯一性。
- 校验两个模块同一天学习时进度、会话、每日目标和导出记录完全隔离。
- 校验旧六级本地记录自动迁移且内容不变。
- 校验六级和考研云端类名、查询和写入互不交叉。
- 校验 35 个考研 WAV 分块、3500 条索引、采样格式和随机单词非静音。
- 校验 Android 构建不包含大型音频目录，资源清单能保留未变化哈希并只清理旧版本分块。
- 校验更新界面同时显示当前版本与目标版本。
- 运行完整自动化测试、网页构建、Capacitor 同步与 GitHub Android 构建。
- 发布后在线验证版本清单、两个模块的随机音频包和 APK 下载地址。

## 成功标准

用户可在首页明确选择六级或考研模块；任一模块的学习操作都不会改变另一模块的数据；两个模块均可在华为手机和网页版按需播放英式女声；老用户升级后原有六级记录完整保留并继续同步；今后的内容更新只传输新增或变化的资源。
