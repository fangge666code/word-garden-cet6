# 个人稳定乱序与学习状态导出设计

## 目标

让不同用户获得不同的六级单词乱序，同时保证同一用户在刷新和重新打开网站后顺序不变。调整每日队列为新词优先，并新增 Excel/WPS 可读的学习状态表格导出。

## 个人稳定乱序

- 用户首次打开网站时生成一个随机种子，只保存在该浏览器的本地学习数据中。
- 单词顺序由“随机种子 + 单词 ID”计算，结果可重复。
- 不同浏览器通常拥有不同种子，因此用户之间顺序不同。
- 同一用户刷新或重新打开网站后顺序不变。
- JSON 备份包含随机种子；恢复备份后保持原顺序。
- 旧数据没有种子时自动补充，不清空学习进度。
- 清除浏览器数据或未恢复备份地更换设备后会产生新顺序。

词库页面、搜索结果和新词选择均使用个人顺序。搜索只过滤结果，不重新洗牌。

## 新词优先队列

每日学习队列按照以下顺序组成：

1. 当天尚需完成的新词；
2. 已到复习时间的旧词；
3. 用户选择“忘记了”后插入的重试词。

到期复习词仍按照复习时间先后排列。重试词沿用现有规则，在至少三张卡片后重新出现。

## 可读表格导出

保留现有 JSON 导出，继续用于完整备份和恢复。新增“导出学习状态表格”按钮，生成带 UTF-8 BOM 的 CSV 文件，可直接使用 Excel 或 WPS 打开。

### 导出范围

只导出已经存在学习进度记录的单词。完全未接触的单词不生成行。

### 状态映射

- `mastered`：已掌握；
- 最近评分为 `forgot` 且尚未掌握：未了解；
- 其他已接触但未掌握的单词：学习中。

### 表格列

1. 单词；
2. 词性；
3. 中文释义；
4. 学习状态；
5. 最近复习时间；
6. 下次复习时间；
7. 累计评分次数。

时间使用用户本地时区的可读格式。字段按照 CSV 规则处理逗号、引号和换行，避免列错位。没有已接触单词时不下载空文件，并显示提示。

## 兼容与错误处理

- 数据版本保持兼容，旧记录加载时补充随机种子。
- 导入旧 JSON 后若缺少种子则生成新种子；导入新 JSON 则保留其中种子。
- 浏览器不支持随机 UUID 时使用本地随机值回退。
- CSV 生成失败时提示用户，不影响学习记录。

## 验证

- 相同种子生成相同顺序，不同种子生成不同顺序。
- 新词位于复习词之前，重试逻辑保持正常。
- 词库页面顺序与新词选择使用同一用户种子。
- CSV 只包含已接触单词，状态映射正确。
- CSV 中文、逗号、引号和日期在 Excel/WPS 中可读。
- JSON 导入导出和现有学习进度保持正常。
