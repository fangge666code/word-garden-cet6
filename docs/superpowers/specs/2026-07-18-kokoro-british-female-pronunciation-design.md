# Kokoro 英式女声发音升级设计

## 目标

把当前较粗糙的 eSpeak NG 离线发音替换为更自然、清晰的英式女声。网页版和 Android App 都必须继续支持无需系统语音包的单词发音，并保留现有学习记录、账号同步和更新流程。

## 方案

采用开源 Kokoro-82M 的英式女声 `bf_emma`，在发布前一次性生成 3000 个 CET-6 单词的音频。模型只用于开发阶段生成素材，不随网站或 App 分发。生成后的单词音频会去除多余静音、统一响度，并转换为兼顾音质和体积的单声道 PCM。

继续沿用现有的 30 个音频分块和单词索引结构：每 100 个单词合并为一个 WAV 文件，网页按需下载当前分块，Android 安装包内置全部分块。Web Audio API 仍是首选播放器，浏览器语音与 Android 原生 TTS 作为备用方案。

## 音质与体积

- 声音：Kokoro `bf_emma`，英式女声。
- 目标格式：16000 Hz、16 位、单声道 PCM WAV；若完整素材明显超过预期体积，则在不影响单词辨识度的前提下降至 12000 Hz、16 位。
- 单词之间保留短静音，播放时仍按索引精确定位。
- 目标 APK 体积控制在约 50–80 MB；网页版只下载用户实际播放到的音频分块。

## 生成与容错

生成脚本必须支持断点缓存，避免中断后重新生成全部单词。每条音频会验证非空、时长合理、峰值不过载。生成失败的单词会记录并重试，不允许静默写入空音频。

运行时若女声音频下载、解码或播放失败，继续按现有顺序回退到 Web Speech 和 Android 原生 TTS，并向用户显示明确提示。

## 验证

- 校验 3000 个索引均存在且落在对应 WAV 数据范围内。
- 校验 30 个 WAV 的采样率、位深、声道、RIFF/WAVE 结构与有效时长。
- 抽查 `wooden` 等单词的音频不是静音，并确认播放定位正确。
- 运行全部自动化测试、网页构建和 Android 同步/构建。
- 发布新版本后在线验证版本清单、音频响应类型、随机分块及 APK 下载地址。

## 成功标准

华为手机和桌面网页版无需安装英语语音包即可播放清晰的英式女声；原有学习数据不丢失；网页更新后按需获取新音频，Android 用户可选择升级到包含女声音频的新安装包。
