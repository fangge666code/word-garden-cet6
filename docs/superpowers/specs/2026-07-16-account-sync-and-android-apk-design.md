# “词间”账号云同步与安卓 APK 设计

## 目标

为“词间 · CET-6 学习”增加开放注册的用户名密码账号，使同一用户在手机、电脑及不同设备之间继续学习。为华为等不提供 PWA 安装按钮的安卓设备提供可直接下载的 APK，同时保留电脑浏览器现有的 PWA 安装方式。

本次使用 LeanCloud 提供账号和云端数据服务，使用 Capacitor 包装安卓应用，并通过 GitHub Actions 构建、签名和发布 APK。

## 产品形态

- 网页地址和 GitHub Pages 部署保持不变。
- 电脑继续使用现有 PWA，可从 Chrome 或 Edge 安装。
- 安卓用户可从网站设置页下载正式 APK。
- 网页、PWA 和 APK 共用同一套 LeanCloud 账号及学习数据。
- 不上架应用商店；APK 通过 GitHub Releases 公开分发。
- APK 包名固定为 `com.wordgarden.cet6`，后续版本保持相同包名和签名。

## 账号功能

设置页增加账号中心，支持注册、登录、同步状态和退出登录。

### 注册

- 任何人都可以自行注册。
- 使用自定义用户名和密码，不要求邮箱或手机号。
- 用户名全站唯一，只允许字母、数字和下划线，长度为 3–20 位。
- 密码至少 8 位，需要再次输入确认。
- 注册前明确提示：本版本不提供密码找回；忘记密码后只能重新注册。
- 注册成功后自动登录，并询问是否把当前设备的匿名学习记录合并到账号。

### 登录与退出

- 登录成功后读取云端数据，并与该设备当前可迁移的匿名记录合并。
- LeanCloud SDK 管理登录会话；应用不保存或记录明文密码。
- 退出前先尝试完成待同步操作。
- 在线同步成功后才能正常退出，并清除该账号在设备上的本地缓存。
- 如果离线或仍有未同步操作，阻止直接退出并提示用户先联网，避免进度丢失。
- 退出后返回独立的匿名学习空间，不显示上一个账号的数据。

## 云端数据模型

每条云端对象都关联当前用户，并使用仅所有者可读写的 ACL。

### UserProfile

- 用户指针；
- 个人乱序种子；
- 每日新词数量；
- 减少动画设置；
- 数据版本；
- 最近同步时间。

### WordProgress

- 用户指针；
- 单词 ID；
- 学习状态；
- 首次学习时间；
- 最近复习时间；
- 下次复习时间；
- 连续认识次数；
- 累计评分次数；
- 最近评分；
- 记录更新时间。

同一用户与单词 ID 的组合保持唯一。只为已经接触的单词创建记录，未接触词不上传。

### DailyRecord

- 用户指针；
- 本地日期；
- 新词 ID 集合；
- 复习词 ID 集合；
- 评分次数；
- 是否完成；
- 记录更新时间。

同一用户与日期的组合保持唯一。

## 本地数据与同步

- 未登录用户继续使用现有本地存储，可正常试用网站。
- 登录用户使用按 LeanCloud 用户 ID 隔离的本地缓存。
- 本地记录始终先保存，云端同步在后台进行。
- 待上传操作写入持久化同步队列，刷新或关闭页面后仍保留。
- 网络恢复、应用启动、用户评分后和手动点击同步时触发同步。
- 设置页显示“已同步”“等待同步”“同步失败”和最后同步时间。

### 合并规则

- 单词记录按 `lastReviewedAt` 比较，保留时间较新的完整记录。
- 如果其中一条记录缺少最近复习时间，保留存在有效时间的记录。
- 时间相同则优先保留累计评分次数更高的记录。
- 每日新词和复习词 ID 使用集合并集。
- 每日评分次数取两端较大值，避免同一记录被重复累加。
- 每日完成状态只要任一端为完成即为完成。
- 个人乱序种子以云端已有值为准；新账号首次合并时使用当前设备种子。
- 设置使用更新时间较新的版本。

同步请求必须可重复执行而不产生重复记录。失败时保留本地数据与待同步队列，稍后重试。

## 安全与权限

- 账号密码只通过 LeanCloud SDK 传输和验证。
- LeanCloud 应用 ID 与客户端 Key 可写入前端配置；主密钥不得出现在仓库、网页或 APK 中。
- `UserProfile`、`WordProgress` 和 `DailyRecord` 均使用当前用户专属 ACL。
- 所有查询同时限定当前用户，不能依赖客户端界面隐藏实现权限隔离。
- 用户输入进入 HTML 前继续使用现有转义规则。
- 登录和注册错误使用中文提示，不展示内部堆栈或服务密钥。
- 公共注册接受 LeanCloud 的频率限制；如果后续出现滥用，再单独增加验证码或邀请机制。

## 安卓 APK

### Capacitor 包装

- 新增独立安卓工程，复用当前网站构建产物和界面。
- 安卓应用名称为“词间”，使用现有绿色 W 图标。
- APK 内的网页代码与发布时的网站版本一致。
- 登录、云同步和版本检查需要网络；断网时保留本地待同步数据并显示提示。
- 外部下载链接交给系统浏览器，学习页面留在应用内。
- 正确处理安卓返回键、状态栏、安全区域和文件下载。

### 签名与构建

- 生成永久发布签名密钥，后续版本必须复用。
- 签名密钥及密码以 Base64 和 GitHub Secrets 保存，不提交到仓库。
- GitHub Actions 在版本标签发布时安装依赖、构建网页、同步 Capacitor、构建签名 APK 并创建 GitHub Release。
- Release 文件名保持稳定，例如 `word-garden-android.apk`。
- 网站下载按钮指向最新 Release 的稳定下载地址。
- 设置页显示当前 APK 版本，并提供检查新版本入口。

## 安装与升级

- 华为用户点击网站中的“下载安卓版 APK”按钮获取文件。
- 首次安装时系统可能要求允许当前浏览器“安装未知应用”，页面提供步骤说明。
- 安装包、包名和签名一致时，新版本可以覆盖升级并保留本地数据。
- 电脑端继续显示 PWA 安装区域，不替换为 APK 下载。
- 安卓网页端同时显示 APK 下载和添加到桌面说明，用户可自行选择。

## 实施分期

本设计分为两个可独立验收的子项目，按顺序实施：

1. 账号与云同步：先完成 LeanCloud 配置、注册登录、数据隔离、迁移合并和多设备同步，并部署网页版本；
2. 安卓 APK：在网页账号同步稳定后，再加入 Capacitor 安卓工程、永久签名、GitHub Actions Release 和网站下载入口。

第二阶段复用第一阶段的账号与同步模块，不另建数据协议。第一阶段未通过跨设备验收前不开始公开发布 APK，避免多个客户端同时放大同步缺陷。

## 配置前提

实施需要：

1. 创建 LeanCloud 应用，获得应用 ID、客户端 Key 和服务地址；
2. 配置 LeanCloud 用户开放注册和所需数据类权限；
3. 生成安卓永久签名密钥；
4. 把签名密钥、别名和密码保存为 GitHub Actions Secrets；
5. 确认 GitHub Actions 有权限创建 Releases。

这些外部配置需要使用账号所有者权限完成。配置值中的客户端公开标识可以进入项目，主密钥和签名秘密只进入 GitHub Secrets。

## 错误处理

- LeanCloud 不可用：继续保存本地进度，显示等待同步。
- 用户名重复或格式错误：保留表单内容并显示明确提示。
- 密码错误：只提示用户名或密码不正确，避免泄露账号是否存在。
- 云端数据格式异常：跳过异常记录、保留本地数据并记录非敏感诊断信息。
- 登录时匿名数据与云端冲突：按既定合并规则处理，并在成功后展示合并结果摘要。
- APK 下载失败：保留网页学习入口并提示重新下载。
- APK 版本检查失败：不阻止学习。

## 验证

### 自动测试

- 用户名、密码和确认密码校验；
- 匿名数据到账号数据的迁移；
- 单词、每日记录、设置和乱序种子的合并规则；
- 重复同步的幂等性；
- 离线待同步队列的持久化和恢复；
- 退出登录清理账号缓存；
- LeanCloud ACL 和查询所有者条件；
- 原有学习、导入导出、PWA 和 1500 词库测试保持通过。

### 联调验收

- 两台设备使用同一账号继续学习；
- 两台设备修改不同单词后正确合并；
- 同一单词冲突时保留最近记录；
- 断网学习后恢复网络自动同步；
- 其他账号无法读取当前用户数据；
- 华为手机下载安装 APK、登录、学习、同步和覆盖升级；
- 电脑 Chrome/Edge 继续安装和使用 PWA；
- GitHub Release 下载链接和网站按钮可用；
- APK 与网页均能访问 LeanCloud 服务且无明文秘密泄露。
