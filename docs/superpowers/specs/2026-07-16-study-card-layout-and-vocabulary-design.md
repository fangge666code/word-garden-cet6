# 学习卡片排版与六级词库扩充设计

## 目标

修复学习页中单词、音标、词性、中文释义和例句相互覆盖的问题，并将现有 60 词词库扩充为 1500 个六级核心词。升级后保留当前学习记录和复习逻辑。

## 范围

- 重做学习卡片的信息层级和响应式排版。
- 取消依赖 3D 背面隐藏的翻转实现，改为点击后原位展开内容。
- 支持较长释义、多词性和多义项的自动换行。
- 将词库扩充到 1500 个六级核心词，并采用稳定的乱序学习队列。
- 保留当前每日目标、评分按钮、间隔复习、本地存储、导入导出和词库页面。

不在本次范围内：账号系统、云端同步、商业词书内容复制、音频文件托管和后台管理。

## 学习卡片设计

### 未展开状态

卡片从上到下显示：学习类型标签、英文单词、音标、点击提示。单词保持视觉主角，其他信息与单词之间使用固定的垂直间距。卡片使用普通二维布局，不创建不可见的背面元素。

### 展开状态

点击卡片后，在同一张卡片中显示：

1. 单词和音标；
2. 词性标签；
3. 中文释义区域；
4. 英文例句区域；
5. 记忆评价提示。

释义区域允许自然换行。若一个词有多个词性或义项，数据层使用分号分隔，展示层将其转换为清晰的分项。例句使用独立的浅色区块，与释义保持明显间距。

### 交互

- 第一次点击展开释义，并显示“忘记了 / 有点模糊 / 认识”三个评分按钮。
- 展开后再次点击卡片不折叠，避免误触隐藏答案。
- 空格键仍可展开卡片；数字键仍可提交评分。
- 卡片高度由内容决定，但设置合理的最小高度，避免短内容页面跳动。

### 响应式规则

- 桌面端卡片最大宽度约 720px，内容区域居中。
- 小屏幕降低单词字号、卡片内边距和按钮间距。
- 长单词允许缩小字号或断行，但不得溢出卡片。
- 评分按钮在窄屏保持三列；极窄屏改为单列。

## 词库设计

### 规模与组织

- 总量：1500 个六级核心词。
- 顺序：数据文件保存稳定基础顺序，生成每日学习队列时使用确定性乱序，避免按字母表连续学习。
- 每个词具有稳定 ID，现有 60 个词的 ID 保持不变，从而保留已有进度。

### 数据字段

每个词条包含：

- `id`：稳定唯一标识；
- `word`：英文单词；
- `phonetic`：规范音标；
- `pos`：一个或多个词性；
- `meaning`：简明中文释义；
- `example`：独立编写的六级难度英文例句。

词汇选择借鉴六级核心、高频、学术与阅读常见词的范围，以及乱序和语境学习的组织思想；不逐字复制新东方或其他商业词书的释义、例句、词根联想和编排顺序。

### 数据质量

- 对缺失字段、重复单词、重复 ID 和异常字符进行构建期校验。
- 保证音标、中文和例句均以 UTF-8 保存。
- 词库页面能够搜索新增词条，并继续显示学习状态。

## 数据流与兼容性

应用加载新词库后，按照稳定 ID 合并现有本地学习记录。旧用户的已掌握、复习日期、连续天数和设置不重置；新增词自动处于“未学习”状态。若旧学习队列含有不存在的 ID，则重新生成当天队列并给出提示。

## 错误处理

- 未找到词条时安全重建学习队列。
- 词条字段不完整时构建失败，不发布损坏版本。
- 本地存储数据异常时沿用现有恢复和提示机制。
- 释义过长时仅换行，不截断关键信息。

## 验证

- 单元测试：词库数量、ID 唯一、单词唯一、必填字段、乱序稳定性、旧进度合并。
- 页面测试：未展开与展开状态无重叠；长单词、长释义和移动端布局正常。
- 回归测试：评分、键盘操作、每日目标、搜索、导入导出和复习队列正常。
- 发布验证：本地构建通过后推送 GitHub，并确认 GitHub Pages 展示新版本。
