# “词间”3000 词扩充、英式发音与首页登录设计

## 目标

在不破坏现有用户学习进度、个性化乱序、复习调度和跨设备同步的前提下，将词库从 1500 词扩充到 3000 词，为全部词条增加可点击的英式发音，并把登录注册入口从设置页移到首页。升级后必须登录才能进入学习、词库和设置区域。

本次扩充采用适合大学英语六级的高频核心词汇补充方案，不复制或重排受版权保护的《新东方六级词汇乱序版》全文。词条数据应具有独立整理的单词、音标、词性、中文释义和例句。

## 已确认的产品决策

- 词库总量由 1500 增加至 3000。
- 保留现有 1500 个词条及其 ID，新增约 1500 个不重复词条。
- 保留“不同用户顺序不同、同一用户顺序稳定”的个性化乱序。
- 每日学习队列仍然以新词优先，然后安排到期复习词。
- 发音只在用户点击扬声器按钮时播放，不自动播放。
- 优先使用英式英语语音，只朗读单词，不朗读例句。
- 发音使用浏览器或系统自带语音，不随安装包内置音频文件。
- 登录和注册移动到首页；未登录用户不能进入学习内容。
- 电脑端 PWA 安装方式和安卓 APK 下载方式继续保留。

## 词库扩充与兼容性

### 数据结构

全部词条继续使用现有字段：

- `id`：稳定且唯一的内部标识；
- `word`：英文单词或词组；
- `phonetic`：音标；
- `pos`：词性；
- `meaning`：中文释义；
- `example`：英文例句。

现有 `cet6-001` 至 `cet6-1500` 不做编号变更。新增词条从后续编号继续追加，以保证 Supabase 中已经保存的 `word_id`、本机进度和导入备份仍能正确对应原词。

### 内容规则

- 新增词条面向 CET-6 高频与核心使用场景，去除与现有词库重复的单词。
- 单词、音标、词性、中文释义和例句均不得为空。
- 中文释义应简洁并按常用义项编排，避免释义和单词重叠显示。
- 例句采用可理解的原创或许可来源内容，不复制整本商业词书。
- 词库生成后执行总数、唯一 ID、唯一单词、字段完整性和编码检查。
- 自动验证升级前 1500 个 ID 及单词映射未发生变化。

### 学习顺序

新增词条自动参与现有用户指纹或账号种子生成的稳定乱序。乱序结果不能写死为全体用户共享顺序，也不能在同一用户每次刷新时重新随机。

每日队列继续遵循：

1. 先取该用户尚未接触的新词；
2. 再加入已经到期的复习词；
3. 用户评分后更新掌握状态与下次复习时间；
4. 从未接触的单词不进入学习状态导出结果。

## 发音功能

### 交互

- 学习卡片的单词旁显示扬声器按钮。
- 词库列表中每个单词也提供扬声器按钮。
- 点击按钮只触发朗读，不翻转卡片、不切换词条、不写入学习状态。
- 按钮提供中文无障碍标签，例如“播放 abandon 的英式发音”。
- 连续点击不同单词时，先停止上一次朗读，再播放新单词，避免声音重叠。

### 语音选择

新增独立发音模块封装浏览器 `speechSynthesis` 能力：

1. 优先选择语言为 `en-GB` 的设备语音；
2. 如果没有英式语音，回退到其他英语语音；
3. 如果设备不支持语音合成，禁用发音按钮并显示友好说明；
4. 语音不可用不应影响学习、评分、同步或导出。

语音由用户设备决定，因此不同手机或电脑的音色可能不同；产品只保证优先选择英式英语，不保证所有设备使用完全相同的声音。

## 首页登录与访问控制

### 未登录首页

首页成为账号入口，包含品牌介绍、登录与注册表单。未登录时：

- “学习”“词库”“设置”等受保护入口不可直接使用；
- 访问受保护的哈希路由时自动返回首页账号区域；
- 页面显示“登录后开始学习”的明确提示；
- 网络或登录失败时仍停留在首页，不把失效的本机会话当作已登录。

### 登录后首页

登录成功后，首页切换为个人学习总览，显示：

- 当前用户名与同步状态；
- 今日新词、复习和完成情况；
- “开始学习”主按钮；
- 手动同步与退出账号入口；
- 电脑安装和安卓 APK 下载入口继续可访问。

设置页面移除登录注册表单，只保留每日新词数量、减少动画、数据导入导出和其他学习偏好。

### 本机旧记录迁移

升级前设备可能已经保存匿名学习记录。用户首次成功注册或登录时继续显示迁移选择：

- “合并本机记录”：按照既有合并规则把本机已接触词条加入账号并同步；
- “只使用账号记录”：不把本机匿名记录写入账号，以云端账号数据为准。

任何选择都不得把尚未接触的 3000 词整体写入学习记录。

### 账号与开发者可见信息

Supabase Authentication 保存账号身份，开发者可在后台查看用户 ID、内部认证邮箱、用户名元数据、创建时间和最近登录时间。学习数据分别保存在 `user_profiles`、`word_progress` 和 `daily_records`，通过 `user_id` 关联。

密码不得写入业务表、日志、导出文件或前端存储。Supabase 只保存不可逆密码哈希；开发者不能查看或还原用户的原始密码，只能执行重置或删除账号等管理操作。

## 模块边界

- 词库数据保持为独立数据模块，并增加词库校验脚本或测试。
- 发音逻辑放入独立模块，负责能力检测、语音选择、播放与停止。
- 路由访问控制封装为统一判断，避免每个页面分别实现不同登录逻辑。
- 首页负责账号入口和登录后的今日概览，不直接实现底层 Supabase 请求。
- 既有认证与同步模块继续负责会话、迁移、上传、下载及冲突合并。

## 异常处理

- 无可用英式语音：回退到其他英语语音。
- 无语音合成功能：显示不支持提示，其他功能继续工作。
- 登录失败：保留表单内容并显示通用中文错误，不泄露账号是否存在。
- 会话失效：清理无效登录状态并返回首页，保留安全的本机待同步数据。
- Supabase 暂时不可用：不解锁受保护页面；提示稍后重试。
- 同步失败：保留本机记录和待同步队列，不覆盖有效云端数据。
- 新词条校验失败：构建或测试失败，禁止发布不完整词库。

## 验证与验收

### 自动验证

- 词条总数严格为 3000。
- 所有 ID 唯一，所有规范化单词唯一。
- 所有必填字段非空且字符编码正确。
- 原 1500 个 ID 与单词映射保持不变。
- 发音模块优先选择 `en-GB`，并能回退、停止上一次朗读和处理不支持状态。
- 未登录时学习、词库和设置路由均返回首页。
- 登录成功后首页与受保护路由正常可用。
- 首次登录的本机记录迁移仍可选择合并或忽略。
- 个性化稳定乱序、新词优先、复习调度、同步与导出测试继续通过。

### 人工验收

- 电脑 Edge/Chrome、安卓浏览器和 APK 中均能点击播放单词。
- 至少在有英式系统语音的设备上确认使用英式语音。
- 快速连续点击两个单词不会同时朗读。
- 未登录直接打开 `#study`、`#words` 或 `#settings` 会返回首页。
- 注册、登录、迁移本机记录、跨设备同步和退出账号均可完成。
- 学习卡片在桌面端和手机端排版清晰，单词、音标、释义和例句不重叠。

## 发布

完成实现和测试后：

1. 构建生产网页并更新 GitHub Pages；
2. 同步相同网页资源到 Capacitor 安卓项目；
3. 提升应用版本并生成使用原签名的新版 APK；
4. 通过现有 GitHub Actions 与 Release 流程发布；
5. 检查公开网站、PWA 安装和 APK 下载入口；
6. 保证升级安装不会清除本机学习数据。
